关于go mod
go mod init exercise1  在当前指定目录位置下创建mod文件



一.go mod tidy 
自动清理和补全 go.mod 和 go.sum 文件，确保依赖干净且准确。
具体作用：

1.加：补全依赖
如果你代码里用了某个包，但 go.mod 里没有写，go mod tidy 会自动把它加上。

2.减：清理无用依赖
如果 go.mod 或 go.sum 里写着某个包，但代码中已经没用了，它就会把这些删掉

3.查：同步版本信息
会校验依赖版本、哈希，修复 go.sum 中缺失或错误的信息，保证模块安全一致。

二.go clean -modcache
用途： 清理本地的模块缓存（$GOPATH/pkg/mod）
场景： 遇到依赖包损坏、拉包失败、环境混乱时。

类似“清空缓存重启”那种骚操作，常用来解决离谱 bug。

三.go doc <package> 或 go doc <package>.<FuncName>
用途： 查看包或函数的文档说明（直接在终端）
场景： 不想打开浏览器查文档、写代码时快速查看 API。
很像 Python 的 help()，能立刻看到官方注释，轻便又快。

四.go test ./...
用途： 运行当前项目下所有包的测试
场景： 想一键验证项目是否通过所有单元测试。
./... 代表递归当前目录下所有子包，非常适合大型项目一把梭！

五.go list -m all
用途： 查看项目所有依赖模块（含版本）
场景： 想了解项目用了哪些库，版本是否一致。
类似 npm list，对排查依赖冲突、确认版本特别有用。
